## Requirements

### Requirement: Advance reimbursement ledger storage

The system SHALL store advance reimbursement records in a Google Sheets tab named "代墊還款" with columns: 成員 (member name), 金額 (amount), 日期 (date), 備註 (notes). This ledger is append-only and is independent from the "成員結算" (profit settlement) sheet.

#### Scenario: Ledger sheet created on first use

- **WHEN** an advance reimbursement record is added or read and the "代墊還款" sheet does not exist
- **THEN** the system creates the sheet with headers 成員, 金額, 日期, 備註

### Requirement: Add advance reimbursement record

The system SHALL provide an API action `addAdvanceReimbursement` that appends one row to the "代墊還款" sheet. The payload SHALL contain member and amount, and MAY contain date and notes. Amount MUST be greater than zero.

#### Scenario: Add a reimbursement

- **WHEN** a POST request with action `addAdvanceReimbursement` and payload member "柏文", amount 7665, date "2026-06-05", notes "六月還款" is sent
- **THEN** a new row [柏文, 7665, 2026-06-05, 六月還款] is appended to the "代墊還款" sheet and the response is success

#### Scenario: Reject invalid amount

- **WHEN** a POST request with action `addAdvanceReimbursement` is sent with no member or with amount ≤ 0
- **THEN** the system returns `{ success: false, error: "成員和金額為必填" }` and appends no row

### Requirement: Get advance reimbursement records

The system SHALL provide an API action `getAdvanceReimbursements` that returns all rows from the "代墊還款" sheet as an array of objects with member, amount, date, and notes fields.

#### Scenario: Retrieve all reimbursements

- **WHEN** a GET request with action `getAdvanceReimbursements` is sent
- **THEN** the system returns all reimbursement records; if the sheet does not exist it returns an empty array

### Requirement: One-time historical advance ledger migration

The system SHALL provide a one-time migration action `migrateAdvanceLedger` that corrects production data left in an inconsistent state by the superseded automated-correction approach. The migration SHALL be append-only with respect to manually entered data: it MAY delete rows that were generated by the previous automated correction (identifiable by note "代墊還款修正" with an incorrect amount), but it MUST NOT delete or alter the manually entered 4-month and 6-month settlement rows.

The migration SHALL:

1. Remove the two incorrect machine-generated settlement correction rows: 柏文 −15,745 and 又又 −14,380 (note "代墊還款修正").
2. Append correct profit-only settlement correction rows so that profit settlement totals reach the confirmed values: 柏文 −7,665 and 又又 −8,380.
3. Leave the already-correct correction rows untouched: 芭樂 −1,600 and 兔子 −6,000.
4. Seed the "代墊還款" ledger with the historically-reimbursed amounts: 柏文 7,665, 又又 8,380, 芭樂 1,600, 兔子 6,000, each noted as "歷史代墊還款".
5. Be idempotent: if the "代墊還款" sheet already contains rows noted "歷史代墊還款", the migration makes no changes and reports `already-migrated`.

After migration, the profit settlement total per member SHALL be: 柏文 30,000, 又又 30,000, 芭樂 30,000, 兔子 30,000, 巧達 30,000, 大弋 69,850, 竹節蟲 69,850, 傑哥 0.

#### Scenario: First migration run corrects data

- **WHEN** `migrateAdvanceLedger` is invoked on production data where 柏文's settlement total is 21,920 (including a −15,745 machine correction) and the "代墊還款" sheet has no "歷史代墊還款" rows
- **THEN** the −15,745 row is removed, a −7,665 row is appended, 柏文's settlement total becomes 30,000, a 代墊還款 row of 7,665 is seeded for 柏文, and the manually entered 20,000 and 17,665 settlement rows remain unchanged

#### Scenario: Migration is idempotent

- **WHEN** `migrateAdvanceLedger` is invoked and the "代墊還款" sheet already contains rows noted "歷史代墊還款"
- **THEN** the migration makes no changes and returns status `already-migrated`

##### Example: post-migration member state

- **GIVEN** pre-migration settlement and advance data per the design migration table
- **WHEN** `migrateAdvanceLedger` runs once
- **THEN** the resulting per-member state is:

| 成員 | 成員結算加總 | 代墊還款帳本 | 代墊總額 | 代墊未結清 |
| ---- | ----------- | ----------- | ------- | --------- |
| 柏文 | 30,000 | 7,665 | 15,745 | 8,080 |
| 又又 | 30,000 | 8,380 | 14,380 | 6,000 |
| 芭樂 | 30,000 | 1,600 | 1,600 | 0 |
| 兔子 | 30,000 | 6,000 | 6,000 | 0 |
