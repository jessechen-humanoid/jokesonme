## Requirements

### Requirement: Automatic tax reserve expense per show and allocation group

The system SHALL automatically maintain "稅務預留" expense transactions for each show. For every show, the system SHALL group all income transactions by their `excludedMembers` value, and for each distinct group SHALL maintain exactly one auto-generated tax reserve expense transaction. The amount of each such expense SHALL equal the sum of the group's income × `TAX_RESERVE_RATE` (default 0.03), stored as a negative number. The expense SHALL have `category = "稅務預留"`, `autoGenerated = true`, and `excludedMembers` equal to the group's `excludedMembers` value.

#### Scenario: Show with only common income

- **WHEN** a show has three income transactions with `excludedMembers = ""` totalling $30,000
- **THEN** the system maintains exactly one auto-generated expense with `category = "稅務預留"`, `amount = -900`, `excludedMembers = ""`, `autoGenerated = true`

#### Scenario: Show with mixed allocation groups

- **WHEN** a show has $10,000 of income with `excludedMembers = ""` and $8,000 of income with `excludedMembers = "芭樂"`
- **THEN** the system maintains two auto-generated tax reserve expenses: one with `amount = -300, excludedMembers = ""` and one with `amount = -240, excludedMembers = "芭樂"`

#### Scenario: Show with no income

- **WHEN** a show has no income transactions
- **THEN** the system maintains zero auto-generated tax reserve expenses for that show

### Requirement: Tax reserve recalculation on transaction writes

The system SHALL recalculate the tax reserve expenses for the affected show whenever a transaction write operation (`addTransaction`, `updateTransaction`, `deleteTransaction`) completes. Recalculation SHALL delete all existing auto-generated tax reserve transactions for that show and insert new ones based on the current income state.

#### Scenario: Adding a new income triggers recalculation

- **WHEN** the user adds a new income transaction of $5,000 with `excludedMembers = ""` to a show that previously had $30,000 common income
- **THEN** the auto-generated tax reserve expense for `excludedMembers = ""` is recalculated to `amount = -1050`

#### Scenario: Deleting the last income of a group removes its tax reserve

- **WHEN** the user deletes the only remaining income transaction in an allocation group
- **THEN** the auto-generated tax reserve expense for that group is removed

#### Scenario: Changing income amount recalculates without adding a new row

- **WHEN** the user edits an existing income transaction's amount from $10,000 to $12,000
- **THEN** the corresponding auto-generated tax reserve expense's amount updates, and no new tax reserve row is created

### Requirement: Auto-generated tax reserve is read-only

The system SHALL prevent users from editing the amount of, or deleting, any transaction where `autoGenerated = true` and `category = "稅務預留"`. The backend SHALL reject such mutation requests with an explanatory error. The frontend SHALL display a "自動" tag on these transactions and disable the edit and delete controls.

#### Scenario: User attempts to delete an auto-generated tax reserve

- **WHEN** the user attempts to delete a transaction with `autoGenerated = true` and `category = "稅務預留"`
- **THEN** the system rejects the request and does not remove the transaction

#### Scenario: User attempts to edit the amount of an auto-generated tax reserve

- **WHEN** the user attempts to change the amount of a transaction with `autoGenerated = true` and `category = "稅務預留"`
- **THEN** the system rejects the amount change

#### Scenario: Transaction list displays auto tag

- **WHEN** the transaction list renders a transaction with `autoGenerated = true`
- **THEN** a "自動" tag is shown next to that row and the edit/delete buttons are visually disabled

### Requirement: Centralised tax rate constant

The system SHALL define the tax reserve rate as a single named constant `TAX_RESERVE_RATE` in the backend (default value 0.03). All tax reserve calculations SHALL reference this constant. Changing the rate SHALL require modifying only this single location in code.

#### Scenario: Rate change propagates everywhere

- **WHEN** the `TAX_RESERVE_RATE` constant is changed from 0.03 to 0.05 and any transaction write occurs
- **THEN** all auto-generated tax reserve expenses for the affected show are recalculated using 0.05

### Requirement: One-time migration for existing shows

The system SHALL provide a one-time migration procedure that scans every existing show, groups its existing income transactions by `excludedMembers`, and creates the corresponding auto-generated tax reserve expenses. After migration, every show with income SHALL have the correct set of auto-generated tax reserve expenses.

#### Scenario: Migration populates all shows

- **WHEN** the migration is executed on a sheet containing N shows with income
- **THEN** every show with income has auto-generated tax reserve expense(s) matching its income groups, and shows without income have none
