## API Design

### getForecast

讀取「財務預估」工作表所有區塊，回傳結構化 JSON。

```
GET ?action=getForecast

Response:
{
  "success": true,
  "data": {
    "baseParams": [
      { "row": 4, "label": "大廳場租 / 場", "value": 16000 },
      ...
    ],
    "versionParams": {
      "headers": ["保守版", "基礎版", "激進版"],
      "rows": [
        { "row": 22, "label": "大廳場次", "values": [11, 11, 11] },
        ...
      ]
    },
    "income": { ... },
    "expense": { ... },
    "pnl": { ... },
    "profitShare": { ... }
  }
}
```

### updateForecast

批次寫回可編輯參數。

```
POST ?action=updateForecast
Body:
{
  "baseParams": [
    { "row": 4, "value": 18000 },
    ...
  ],
  "versionParams": [
    { "row": 22, "values": [11, 12, 11] },
    ...
  ]
}
```

寫回後 Sheet 公式自動重算，前端再呼叫 `getForecast` 取得最新結果。

## UI Layout

```
┌─────────────────────────────────┐
│  財務預估                        │
├─────────────────────────────────┤
│  ① 分潤規劃（唯讀 card）         │
│  三欄：保守版 / 基礎版 / 激進版    │
│  共同基金 / 成員分潤 / 每人分潤    │
├─────────────────────────────────┤
│  ② 基礎參數（可編輯 form）       │
│  每行：參數名 + input            │
├─────────────────────────────────┤
│  ③ 版本參數（可編輯 table）      │
│  三欄 input：保守 / 基礎 / 激進   │
├─────────────────────────────────┤
│  [儲存參數] 按鈕                  │
├─────────────────────────────────┤
│  ④ 損益彙總（唯讀 card）         │
│  年度淨利 / 淨利率 × 3 版本       │
├─────────────────────────────────┤
│  ⑤ 收入計算（唯讀 table）        │
├─────────────────────────────────┤
│  ⑥ 支出計算（唯讀 table）        │
└─────────────────────────────────┘
```

## Data Flow

```
[儲存] → updateForecast (GAS) → 寫回 Sheet
       → getForecast (GAS) → 讀取重算結果
       → 前端更新唯讀區塊
```

## Edge Cases

- Sheet 欄位為空時：顯示 0
- 百分比欄位（票房售出率、流失率）：前端顯示/輸入用 %，存回 Sheet 時轉為小數或維持原格式
- 儲存中：按鈕 disable + loading 狀態，防止重複送出
