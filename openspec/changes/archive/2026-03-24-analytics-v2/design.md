## Context

財務分析頁面目前使用進度條顯示收入/支出佔比，有獨立的代墊追蹤區塊，成員報表只顯示預估淨收入。需升級為圓餅圖視覺化、整合代墊資訊、加入結算功能。

## Goals / Non-Goals

**Goals:**
- 圓餅圖視覺化收入/支出佔比
- 移除獨立代墊追蹤，整合至成員報表
- 成員報表 5 欄完整呈現年度收支
- 支援分次結算匯款紀錄

**Non-Goals:**
- 不做自動匯款（只記錄）
- 不做成員個人頁面

## Decisions

### 使用 Canvas API 繪製圓餅圖

用原生 Canvas API 繪製，不引入圖表庫（如 Chart.js），保持零依賴的架構風格。

- 替代方案：Chart.js → 增加 ~200KB 依賴
- 替代方案：純 CSS conic-gradient → 無法加標籤和圖例
- 選擇理由：Canvas API 足夠畫圓餅圖，不需外部庫

### 成員結算紀錄存在新 Sheet tab

新增「成員結算」sheet tab，欄位：成員、金額、日期、備註。

- 替代方案：用現有交易紀錄加特殊分類 → 語義不清，且交易紀錄是按專案分的
- 選擇理由：結算是跨專案的成員級操作，獨立 sheet 更清楚

### 結算 UI 用 modal 表單

在成員報表區塊加「新增結算」按鈕，點擊後彈出 modal，選擇成員、輸入金額、日期、備註。

### 成員報表計算邏輯

- 年度淨利 = 該成員分配到的收入 - 均攤支出（現有邏輯）
- 已收款淨利 = 該成員所有結算紀錄的金額總和
- 未收款淨利 = 年度淨利 - 已收款淨利
- 代墊未結清 = 該成員所有未結清的代墊金額
- 需匯款金額 = 未收款淨利 + 代墊未結清

## Risks / Trade-offs

- [取捨] Canvas 圓餅圖需要手動處理標籤和圖例，但避免了引入圖表庫
- [風險] 結算金額如果超過年度淨利會導致已收款 > 年度淨利 → UI 上用顏色提示
