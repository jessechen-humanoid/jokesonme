## Context

目前的看我笑話工作室財務管理平台有四個頁面，需要對外展示其中三個（收支紀錄、財務分析、演出準備）的功能。展示對象為團隊外部人員，因此不能暴露真實財務數字。

## Goals / Non-Goals

**Goals:**

- 一個獨立的 `demo.html` 自包含頁面，不依賴任何外部 API
- 複用現有 CSS（liquid-glass UI）保持視覺一致性
- 假資料數字看起來合理但與真實數據無關
- 漂浮註解氣泡清楚說明各項功能

**Non-Goals:**

- 不需要任何互動功能（表單提交、新增、刪除等）
- 不做響應式設計的特殊處理（複用現有 CSS 即可）
- 不需要密碼保護

## Decisions

### 單一 HTML 檔案自包含

整個 demo 放在一個 `demo.html` 中，包含 inline CSS（或引用現有 css/style.css）和 inline JS。假資料直接 hardcode 在檔案內。不需要額外的 JS/CSS 檔案。

### Tab 切換模擬 nav bar

複用現有 nav bar 的視覺樣式，但 tab 點擊只做 DOM 切換（顯示/隱藏對應區塊），不做頁面跳轉。

### 漂浮註解氣泡設計

使用 CSS absolute/fixed positioned 的註解卡片，帶有指向線（arrow）指向對應的功能區塊。每個氣泡包含：標題（功能名稱）和 1-2 句說明文字。氣泡使用半透明背景，不遮擋主要內容。

### 假資料策略

- 專案名稱：使用虛構但合理的表演名稱（如「春季公演」「校園巡演」等）
- 成員名稱：使用虛構名稱（如「成員 A」「成員 B」等）
- 金額：使用看起來合理的虛構數字

## Risks / Trade-offs

- [維護成本] → demo 頁面是靜態快照，主站功能更新後 demo 不會自動同步。可接受，因為 demo 用途是概念展示而非即時同步。
