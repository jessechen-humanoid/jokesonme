## Why

看我笑話工作室需要兩個互相關聯的財務邏輯調整：
1. 所有有收入的專案都要預留 3% 作為「稅務預留」，讓每個專案和整體年度淨利反映扣稅後的真實可分配數字。
2. 共同基金要升級成「有餘額概念的虛擬帳戶」——會計費等特定支出直接由基金扣款，避免雙重扣除、讓成員可分配淨利不被這類費用影響。

這兩件事都會改動交易資料模型與分析頁的計算邏輯，合併在同一個 change 裡處理以確保一致性。

## What Changes

- **新增「稅務預留」expense category**（expense 分類從 7 個變 8 個）
- **自動稅務預留支出（A2 方案）**：每個 show × 每個 `excludedMembers` 組合 → 一筆自動支出；金額 = 該組收入總和 × 3%；`autoGenerated=true`；不可手動編輯 / 刪除
- **自動稅重算邏輯**：GAS 後端在 `addTransaction` / `updateTransaction` / `deleteTransaction` 的尾端重算該 show 的自動稅支出（整組刪除重建）
- **稅率常數化**：`TAX_RESERVE_RATE = 0.03` 集中在 GAS 常數區，方便未來調整
- **一次性 migration**：掃描所有既有 show × excludedMembers 組合，產生對應自動稅支出
- **新增交易欄位 `paidByFund: boolean`**（僅支出可為 true）
- **新增交易欄位 `autoGenerated: boolean`**（標記自動稅支出）
- **基金支付語義**：`paidByFund=true` 的支出不計入共同支出、不參與成員分配、`excludedMembers` 強制空
- **新增虛擬 show「共同基金支出」**（default show list 從 13 個變 14 個）；此 show 下的支出 `paidByFund` 預設 true
- **新增一筆年度會計費交易**（$13,000，category=行政雜支，show=共同基金支出，paidByFund=true，notes=「年度會計費用」）
- **交易表單 UI**：支出模式下新增「由共同基金支付」checkbox；勾選時隱藏 excludedMembers 區塊
- **交易列表 UI**：自動稅支出顯示「自動」tag、停用編輯 / 刪除按鈕
- **財務分析頁共同基金區塊擴充**：新增「已動用（Σ paidByFund 支出）」、「基金餘額（累積提撥 − 已動用）」兩行
- **支出分類佔比**：稅務預留作為獨立分類呈現
- **成員年度分配淨利 / 需匯款金額**：計算邏輯不變（paidByFund 不進入分配、稅務預留透過既有 excludedMembers 邏輯自動作用）

## Non-Goals

- 不做會計費的「月度自動重複產生」機制（YAGNI；目前只有一筆，直接手動建一筆年度合計）
- 不做稅率的 UI 設定介面（常數改 code 即可；未來需要再做）
- 不回填「過去由基金支付但還沒標記」的支出（使用者確認過去沒有這類紀錄）
- 不改動「結清狀態」「墊款人」「Opentix 匯入」等其他既有流程

## Capabilities

### New Capabilities

- `tax-reserve-calculation`: 依 show × excludedMembers 分組自動產生與維護稅務預留支出的規則與重算時機
- `common-fund-payment`: 透過 `paidByFund` 欄位標記「由共同基金支付」的支出、其與共同淨利計算的隔離規則、基金餘額計算

### Modified Capabilities

- `transaction-categories`: 新增「稅務預留」expense category（從 7 → 8）
- `transaction-management`: 新增 `paidByFund` 與 `autoGenerated` 欄位；自動稅支出的唯讀規則；交易寫入時觸發稅重算；支出表單新增「由共同基金支付」checkbox
- `show-management`: default show list 從 13 → 14，新增「共同基金支出」虛擬 show
- `financial-analytics`: 共同基金區塊新增「已動用」「基金餘額」；支出分類佔比包含稅務預留
- `income-allocation`: `paidByFund=true` 的支出 `excludedMembers` 強制空且不參與成員分配
