## Context

看我笑話工作室是 8 人漫才團體，使用 Google Sheets 作為後端資料庫、GitHub Pages 作為前端。目前收支紀錄的「墊款人」欄位是單選下拉，收入與支出共用同一欄位。財務分析頁在沒有資料時顯示空白狀態。

團隊需要：
1. 收入時選擇「分配給哪些人」（多選），而非選墊款人
2. 報表在無資料時仍完整呈現
3. 看到每個成員的年度預估淨收入
4. 密碼保護防止未授權瀏覽

## Goals / Non-Goals

**Goals:**

- 收入交易支援多成員分配 checkbox，預設全員勾選
- Google Sheet 新增欄位儲存排除成員
- 收支明細表合併顯示「分配/墊款」資訊
- 財務分析頁無資料時顯示 0 值報表（非空白狀態）
- 新增成員年度預估淨收入報表
- 所有頁面密碼門保護，sessionStorage 驗證

**Non-Goals:**

- 不做收入按比例分配（目前只支援均分）
- 不做支出的成員分攤設定（支出永遠 8 人均攤）
- 不做歷史資料回溯標記排除成員

## Decisions

### D1: 排除成員儲存格式

Google Sheet 「收支紀錄」新增第 9 欄「排除成員」，存逗號分隔的成員名字（如 `巧達,兔子`）。空值代表全員分配。

理由：預設全員分配是最常見情境，只儲存例外（排除的人）讓大部分紀錄保持空值，減少資料量。

### D2: UI 欄位隨收支類型切換

- **支出模式**：顯示「墊款人」單選下拉（現有行為不變）
- **收入模式**：隱藏墊款人，改顯示 8 人 checkbox grid（預設全勾）

### D3: 明細表合併顯示

收支明細表將「墊款人」欄改為「分配/墊款」欄：
- 收入 + 全員 → 顯示「全員」
- 收入 + 有排除 → 顯示「N/8 人」，hover 展開 tooltip 顯示具體名單
- 支出 + 有墊款人 → 顯示「姓名(墊)」
- 支出 + 無墊款人 → 顯示「—」

### D4: 成員年度預估淨收入公式

每位成員的預估淨收入 = Σ(分配到的收入份額) - 總支出 ÷ 8

其中每筆收入的份額 = 收入金額 ÷ 參與分配人數（被排除的人份額為 0）。支出永遠由 8 人均攤，不受收入分配影響。

### D5: 財務分析無資料時行為

移除 `transactions.length === 0` 的 early return，所有報表函式都照常執行。數值為 0 時正常渲染表格框架。各檔演出損益表只列有交易紀錄的演出（無紀錄時表格為空但框架在）。

### D6: 密碼門機制

所有頁面（index.html、checklist.html、analytics.html）載入時檢查 `sessionStorage.getItem('authenticated')`。未驗證時：頁面內容以 overlay 遮蔽，顯示密碼輸入框。使用者輸入正確密碼 `joke0321` 後，`sessionStorage.setItem('authenticated', 'true')` 並移除 overlay。已驗證的 session 不再要求。

實作位置：`js/shared.js` 新增 `checkAuth()` 函式，各頁面在 DOMContentLoaded 最早呼叫。overlay 為全螢幕 fixed div，z-index 最高。

理由：前端密碼門足以防止隨意瀏覽（團隊內部財務資料，非高機密）。sessionStorage 在關閉瀏覽器分頁後自動失效，無需登出功能。密碼寫死在前端 JS 中，適合小團隊場景。

## Risks / Trade-offs

- **排除成員欄位用逗號分隔**：簡單但如果成員名字含逗號會出問題。目前 8 位成員名字都不含逗號，風險低。
- **支出均攤固定 8 人**：如果未來有人退出或加入需要手動調整 MEMBERS 常數。目前可接受。
- **前端密碼門非真正安全**：密碼存在 JS 原始碼中，技術使用者可繞過。但對團隊內部財務資料的防護程度足夠，不需要後端驗證的複雜度。
